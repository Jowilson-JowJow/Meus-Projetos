Especificação Técnica: Sistema de Gestão de Cadastros (CRUD)Este documento fornece as diretrizes técnicas, arquiteturais e operacionais para a implementação e manutenção do Sistema de Cadastro de Usuários. A aplicação é uma interface gráfica (GUI) baseada no paradigma orientado a eventos, integrando-se a um sistema de gerenciamento de banco de dados relacional (SRGBD) via driver assíncrono.1. Arquitetura do SistemaA aplicação segue uma arquitetura de camada única com acoplamento direto ao banco de dados, utilizando as seguintes tecnologias:Interface Gráfica: tkinter (Toolkit de interface padrão Python).Gerenciamento de Widgets: tkinter.ttk (Extensão temática para componentes avançados como Treeview).Driver de Conectividade: mysql-connector-python (Implementação do protocolo MySQL).Persistência de Dados: MySQL 8.0+.2. Modelagem de Dados (Schema)O esquema do banco de dados deve ser normalizado para garantir a integridade referencial. O sistema opera sobre a tabela db_usuarios dentro do banco db_usuario.2.1 Dicionário de DadosCampoTipoRestriçãoDescriçãoidINTPRIMARY KEY, AUTO_INCREMENTIdentificador único e imutável do registro.nomeVARCHAR(60)NOT NULLNome completo do usuário.emailVARCHAR(60)NOT NULLEndereço de correio eletrônico.enderecoVARCHAR(100)NOT NULLLogradouro completo.2.2 Script de Implementação (DDL)SQLCREATE DATABASE IF NOT EXISTS db_usuario;
USE db_usuario;

CREATE TABLE IF NOT EXISTS db_usuarios (
    id INT NOT NULL AUTO_INCREMENT,
    nome VARCHAR(60) NOT NULL,
    email VARCHAR(60) NOT NULL,
    endereco VARCHAR(100) NOT NULL,
    CONSTRAINT pk_usuarios PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
3. Lógica de Negócio e MétodosO fluxo de execução é controlado por funções de callback vinculadas aos widgets da interface:conectar(): Instancia um objeto de conexão utilizando o padrão Singleton-like para cada chamada, garantindo que os parâmetros de rede (host, user, password) sejam aplicados.carregar_dados(): Implementa o ciclo de atualização da UI. Realiza o parsing dos registros do banco e mapeia os objetos para o componente Treeview.selecionar_usuario(event): Manipula o estado do widget Entry do ID. Devido à restrição de segurança, o campo é definido como readonly. Esta função altera o estado para normal via introspecção de software para permitir o preenchimento automático e retorna ao estado readonly para impedir a manipulação direta pelo usuário.Tratamento de Exceções: O sistema utiliza messagebox para validação de campos vazios e confirmação de exclusão lógica/física, mitigando operações acidentais.4. Requisitos de Instalação e Implantação4.1 Ambiente PythonÉ necessária a instalação da biblioteca de interface entre o Python e o motor do banco de dados:Bashpip install mysql-connector-python
4.2 Configuração de RedePor padrão, o sistema busca o servidor MySQL em localhost:3306. Caso o servidor esteja em um host remoto, as strings de conexão na função conectar() deverão ser parametrizadas.4.3 InicializaçãoPara inicializar o ciclo de vida da aplicação e o loop de eventos (Main Loop), execute:Bashpython <nome_do_arquivo>.py
5. Segurança de DadosO sistema implementa a parametrização de consultas (Prepared Statements) através do driver MySQL. O uso de %s nas queries assegura a sanitização das entradas, prevenindo vulnerabilidades de injeção de SQL.
